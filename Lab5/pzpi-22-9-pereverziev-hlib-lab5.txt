3 ОПИС ВИКОНАНОЇ РОБОТИ


3.1 Опис проєкту
Тема: Програмна система для медичного супроводу пацієнтів офтальмологічного та ортопедичного профілю (OrthoVision)
OrthoVision — це сучасна програмна система, створена для цифровізації процесів у клініках, що спеціалізуються на офтальмології та ортопедії. Проєкт бере свій початок з концепції IoT-пристрою — смарт-окулярів, які фіксують рівень освітлення та нахил голови користувача з метою контролю постави й запобігання погіршенню зору.
Система включає кілька функціональних компонентів: серверну частину з базою даних, веб-інтерфейс для лікарів та адміністраторів клінік, а також мобільний застосунок для пацієнтів. Завдяки цьому реалізовано зручне керування записами на прийом, введенням діагнозів, моніторингом стану пацієнтів, а також збиранням статистики для клінічного аналізу.
Мобільний застосунок дозволяє пацієнтам відслідковувати історію прийомів та отримувати рекомендації від лікарів, а веб-панель для фахівців спрощує щоденну роботу з даними та забезпечує оперативний доступ до медичної інформації.
OrthoVision поєднує IoT-технології, сучасні підходи до веб- та мобільної розробки, а також зручний інтерфейс, створюючи ефективне рішення для підтримки щоденної роботи клінік та покращення якості медичного обслуговування.








3.2 Заплановані та виконані функції
Початкова версія програмної системи OrthoVision охоплює основний функціонал для організації роботи офтальмологічних та ортопедичних клінік. Система розроблялась з урахуванням реальних сценаріїв взаємодії між пацієнтами, лікарями та адміністраторами клінік. Особлива увага приділялася коректній інтеграції між компонентами: серверною частиною, веб-застосунком та мобільним додатком. Нижче подано перелік запланованих функцій та фактичну реалізацію в межах першого релізу.
1) Серверна частина (Go + Fiber)
Реалізовано:
* REST API для взаємодії з мобільним і веб-застосунками;
* база даних PostgreSQL з таблицями для користувачів, лікарів, клінік, записів, відгуків та діагнозів;
* авторизація та автентифікація користувачів з JWT-токенами;
* CRUD-операції для адміністрування записів, клінік, профілів лікарів та пацієнтів;
* окрема рольова модель для лікаря, адміністратора та пацієнта;
* горизонтальне масштабування через Kubernetes (Deployment, LoadBalancer-сервіс);
* контейнеризація через Docker (написано власний Dockerfile);
* підтримка змінних оточення (.env) для конфігурації ключових параметрів сервера (порт, база даних, секрети).


2) Мобільний застосунок (Android, Kotlin, MVVM)
Реалізовано:


* екран входу та реєстрації користувача з перевіркою введених даних;
* збереження токену та авторизована взаємодія з API (через SharedPreferences);
* головна сторінка з вибором клініки, підвантаження списку клінік із сервера;
* вибір лікаря, доступного в обраній клініці;
* запис на прийом до лікаря з обранням дати та часу;
* сторінка з історією прийомів користувача (пацієнта), з деталями записів;
* Bottom Navigation з розділами: клініки, мої записи, профіль;
* архітектура MVVM: ViewModel, Repository, Retrofit;
* інтуїтивно зрозумілий інтерфейс, адаптований під роль пацієнта.
* перегляд статистики зі ІоТ пристрою;
3) Веб-застосунок (Next.js + React)
Функціонал лікаря:
* авторизація та захист маршрутів на основі JWT;
* перегляд усіх записів на прийом лікарем (Appointments);
* редагування статусу прийому (наприклад, «очікує», «прийнято», «завершено»);
* додавання або редагування діагнозу пацієнта безпосередньо в записі;


Функціонал адміністратора:


* перегляд і CRUD для списку клінік, користувачів
* зручний інтерфейс управління через таблиці, фільтри та кнопки дій;
* аналітична панель з статистичними даними по захворюванням в клініках
* адаптивний інтерфейс, побудований з використанням UI-компонентів.


4) Інфраструктура та DevOps
Реалізовано:
* Docker-контейнеризація для всіх компонентів (Go API, PostgreSQL, веб);
* Kubernetes-деплой з окремими Deployment, Service та PVC для кожного модуля;
* автоматичне горизонтальне масштабування для ortho-backend за рахунок кількох реплік;
* LoadBalancer для розподілення навантаження між подами;
* повноцінне підключення бази даних до кластеру через ClusterIP;


5) IoT-пристрій (Smart-окуляри на базі ESP32)
Реалізовано:
* Прототип smart-окулярів з сенсорами освітлення та нахилу голови (акселерометр);
* підключення до Wi-Fi (через ESP32);
* зчитування даних з сенсорів освітленості та акселерометра;
* надсилання зібраної інформації на сервер через HTTP-запити (REST API);
* дані з пристрою інтегруються в загальну систему — можуть бути використані лікарями для створення рекомендацій або діагностичних висновків;
* орієнтація на подальшу інтеграцію з мобільним застосунком для персональних рекомендацій;
* Мета розробки пристрою — покращення ергономіки користувача під час роботи за комп’ютером, профілактика проблем зору та опорно-рухового апарату.
Висновок:
Таким чином, було реалізовано повноцінну версію OrthoVision, що включає серверну логіку, мобільний застосунок, веб-інтерфейс та інфраструктурне середовище. Система дозволяє проводити медичні прийоми, керувати записами пацієнтів і вносити діагнози, при цьому залишаючись масштабованою, стійкою до навантажень та зручною для користувачів різних ролей. Створена архітектура демонструє потенціал для подальшого розвитку IoT-складової — смарт-окулярів, які стали основою цієї ідеї.
